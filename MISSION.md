# Mission

**Learner:** Traditional AEM developer (Sites, components, Java/Sling, dialogs, HTL) — new to Edge Delivery Services.

**Goal:** General upskilling — add AEM Edge Delivery Services (EDS / "Franklin" / Helix, document-based authoring) to my toolkit. No fixed deadline or project yet.

**Why it matters:** EDS is Adobe's newer delivery model and increasingly what clients/projects ask for. I want fluency in the parts that differ most from classic AEM.

## What success looks like
- I can stand up an EDS project from the boilerplate and run it locally.
- I understand the authoring→DOM→block pipeline (no JCR, no dialogs, no Java).
- I can build and style a custom block in vanilla JS/CSS.
- I understand content sources (Docs / SharePoint / DA), Sidekick, preview vs. publish.

## Grounding for the teacher
Lean on my existing AEM knowledge — teach by contrast ("in classic AEM you did X; in EDS you do Y"). The big unlearning: no server-side component rendering, no dialogs, no repository. Content is a document; a block is a table; enhancement is client-side JS.
