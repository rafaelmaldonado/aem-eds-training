Sections, Blocks & Default Content

The three-layer anatomy of an EDS page — and which one a table becomes.

In Lesson 1 you saw a table become a block. But a real page is more than blocks: it has plain paragraphs, headings, images, and it's split into bands. This lesson names the three layers EDS builds from a document, so you always know what a given piece of the doc turns into. In classic AEM you thought in parsys → components; here you think in sections → default content + blocks.

The unlearning: Not everything is a component. Most content is default content — headings, paragraphs, lists, links, images rendered straight from the doc with no block, no JS. A block (a table) is the exception, used only when content needs custom structure or behavior. And a section (a --- in the doc) is just a grouping band, like a parsys column — not a component at all.

The three layers

Every EDS page decomposes into exactly these, top to bottom (source):

Layer
What it is in the doc → DOM
Section
A band of content. A --- (horizontal rule) in the doc splits sections. Each becomes a <div class="section">.
Default content
Everything that isn't a table: headings, text, lists, images, links. Rendered as plain semantic HTML — no block, no decorate.
Block
A table whose first cell names it (e.g. Cards). Becomes <div class="cards block"> and gets a decorate() run.

What a section looks like in the DOM

Two paragraphs, a ---, then a Cards block produces two sections — the first is pure default content, the second wraps a block (source):

<main>
  <div class="section">           <!-- section 1: default content -->
    <h1>Welcome</h1>
    <p>Plain paragraph, no block.</p>
  </div>
  <div class="section">           <!-- section 2: after the --- -->
    <div class="cards block">...</div>
  </div>
</main>
Section metadata. A table named Section Metadata inside a section sets attributes on that band (e.g. a Style key adds a class you can target in CSS). It's the closest thing to a layout-container dialog — but it's just another table (source).

The rule of thumb

Reach for a block only when default content can't express what you need — custom layout, grouping, or client-side behavior. If a heading and a paragraph do the job, that's default content and you write no code at all. This is the opposite instinct from classic AEM, where almost everything was a component.
